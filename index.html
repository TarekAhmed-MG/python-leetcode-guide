Language Mechanics

The Python syntax you'll write 100 times in interviews

Variables, Types, Unpacking CORE

Python is dynamically typed — no type declarations. Variables are just names bound to objects. This is unlike Scala's val/var with explicit types, but you should mentally track types anyway.

pythonvariables & unpacking
# Basic assignment — no val/var, no type annotation needed
x = 5
name = "tarek"
flag = True     # capital T, unlike Scala's true

# Multiple assignment — very common in LeetCode
a, b = 1, 2

# Swap WITHOUT a temp variable — idiomatic Python
a, b = b, a     # equivalent to Scala's val (a, b) = (b, a)

# Unpack from a list/tuple
first, second, third = [10, 20, 30]

# Star unpacking — grab head and tail
head, *rest = [1, 2, 3, 4]  # head=1, rest=[2,3,4]
*init, last = [1, 2, 3, 4]  # init=[1,2,3], last=4

# Python has no integer overflow — ints are unbounded
big = 10 ** 18  # works fine, unlike Java's Long.MaxValue concerns
inf = float('inf')   # use for "sentinel max" in DP, Dijkstra etc
neg_inf = float('-inf')
⚡
Scala brain: unbounded ints

In Scala you'd worry about Int.MaxValue overflow. In Python, int is arbitrary precision. Never guard against overflow — it doesn't exist here. Use float('inf') as your sentinel "infinity" value.

Slicing CORE

Slicing is one of Python's most powerful features. Works on strings, lists, anything subscriptable. The syntax is arr[start:stop:step] — all three are optional.

pythonslicing — used constantly
s = "leetcode"
arr = [1, 2, 3, 4, 5]

# Basic slice [start:stop] — stop is EXCLUSIVE
s[0:4]       # "leet"
arr[1:3]     # [2, 3]

# Negative indices — count from end
s[-4:]       # "code" (last 4 chars)
arr[:-1]     # [1, 2, 3, 4] (all except last)

# Reverse — the most used LeetCode trick
s[::-1]      # "edocteel"
arr[::-1]    # [5, 4, 3, 2, 1]

# Step
arr[::2]     # [1, 3, 5] — every other element

# Slicing creates a NEW list — original unchanged
copy = arr[:]  # shallow copy of entire list

# Check palindrome — classic use
def is_palindrome(s):
    return s == s[::-1]

Comprehensions CORE

Comprehensions replace verbose for loops. They're more readable, faster, and idiomatic. You'll use these in almost every medium+ problem.

pythoncomprehensions — list, dict, set
nums = [1, -2, 3, -4, 5]

# List comprehension — equivalent to Scala's .map().filter()
positives = [x for x in nums if x > 0]       # [1, 3, 5]
squares   = [x**2 for x in nums]             # [1, 4, 9, 16, 25]
filtered  = [x*2 for x in nums if x > 0]   # [2, 6, 10]

# Dict comprehension
word = "hello"
freq = {c: word.count(c) for c in set(word)}  # {'h':1,'e':1,'l':2,'o':1}

# Set comprehension
unique_sq = {x**2 for x in [1,2,2,3]}   # {1, 4, 9}

# 2D — build matrix
matrix = [[0] * 3 for _ in range(3)]
# [[0,0,0],[0,0,0],[0,0,0]]
# WARNING: do NOT use [[0]*3]*3 — rows share the same reference!

# Nested comprehension (flatten 2D)
grid = [[1,2],[3,4]]
flat = [x for row in grid for x in row]  # [1,2,3,4]
⚠️
The 2D matrix gotcha

[[0]*3]*3 creates three references to the SAME inner list. Modifying grid[0][0] modifies all rows. Always use [[0]*cols for _ in range(rows)] instead.

Control Flow CORE

pythonloops & control
# enumerate — get index AND value (use this, not range+index)
for i, val in enumerate(['a','b','c']):
    print(i, val)  # 0 a, 1 b, 2 c

# zip — iterate two sequences in parallel
for a, b in zip([1,2,3], ['x','y','z']):
    print(a, b)  # stops at shortest

# range variants
range(5)         # 0,1,2,3,4
range(2, 8)      # 2,3,4,5,6,7
range(10, 0, -1) # 10,9,8...1 (reverse)

# Ternary expression — one-liner if/else
result = "even" if n % 2 == 0 else "odd"
# Scala equivalent: if (n % 2 == 0) "even" else "odd"

# while with else — runs if loop ended naturally (not via break)
while lo <= hi:
    mid = (lo + hi) // 2
    if nums[mid] == target: break
    elif nums[mid] < target: lo = mid + 1
    else: hi = mid - 1

# Integer division and modulo
7 // 2   # 3  (floor division)
7  % 2   # 1
-7 // 2  # -4  ← Python floors towards -inf, unlike Java's truncation!
int(-7 / 2)  # -3  ← use this if you want Java/Scala truncation behaviour
⚠️
Floor division difference from Scala/Java

Python's // floors towards negative infinity. -7 // 2 = -4 in Python but -7 / 2 = -3 in Scala (truncation). For LeetCode with negative numbers, use int(-7 / 2) if you want truncation, or be aware of which you need.

Collections Deep Dive

The six data structures that cover 90% of LeetCode problems

List — Your Default Array CORE

Python's list is a dynamic array. It's what you'll use for stacks, general arrays, and anything indexed. Key: append/pop at the end is O(1), at the front is O(n).

pythonlist — stack usage + key ops
arr = [3, 1, 4, 1, 5]

# O(1) operations
arr.append(9)       # add to end  → [3,1,4,1,5,9]
arr.pop()           # remove end  → returns 9, O(1)
arr.pop(0)          # remove front → O(n) — use deque instead!
arr[-1]             # peek end (no pop)

# Common methods
arr.sort()          # in-place sort, O(n log n)
arr.sort(reverse=True)
arr.sort(key=lambda x: -x)  # sort by custom key
sorted(arr)         # returns NEW sorted list
arr.reverse()       # in-place reverse

# Use list AS a stack (LIFO)
stack = []
stack.append(1)  # push
stack.append(2)
top = stack.pop()  # pop → 2
if stack: peek = stack[-1]  # peek without popping

# Check membership
3 in arr            # O(n) — use set for O(1) lookups

dict — HashMap CORE

Python's dict is your HashMap. All operations — get, set, delete, membership — are O(1) average. This is the most used structure in LeetCode after list.

pythondict — hashmap patterns
freq = {}

# Safe get with default — avoids KeyError
freq.get('a', 0)        # returns 0 if 'a' absent

# Count frequencies — classic pattern
for c in "hello":
    freq[c] = freq.get(c, 0) + 1
# {'h':1, 'e':1, 'l':2, 'o':1}

# Iterate over dict
for key in freq:           # keys only
for val in freq.values():   # values only
for k, v in freq.items():  # both — most common

# Delete a key
del freq['h']
freq.pop('h', None)  # safe delete — won't throw if absent

# Two-sum classic — O(n) with dict
def two_sum(nums, target):
    seen = {}
    for i, n in enumerate(nums):
        complement = target - n
        if complement in seen:
            return [seen[complement], i]
        seen[n] = i
    return []

defaultdict & Counter BUILT-IN

Both from collections. These eliminate boilerplate and are faster than rolling your own frequency maps.

pythondefaultdict + Counter
from collections import defaultdict, Counter

# defaultdict — auto-creates missing keys with a default factory
graph = defaultdict(list)   # missing key → empty list
graph[1].append(2)           # no KeyError — perfect for adjacency lists
graph[1].append(3)

count = defaultdict(int)    # missing key → 0
count['a'] += 1              # no get() boilerplate needed

groups = defaultdict(set)   # missing key → empty set

# Counter — frequency map in one line
c = Counter("aabbbcccc")
# Counter({'c': 4, 'b': 3, 'a': 2})

c.most_common(2)    # [('c',4), ('b',3)] — top-k pattern
c['z']              # 0 — never raises KeyError (unlike dict)

# Counter arithmetic
c1 = Counter("abc")
c2 = Counter("bc")
c1 - c2  # Counter({'a': 1}) — anagram-style subtraction

# Check anagram: are two words anagrams?
Counter("anagram") == Counter("nagaram")  # True

deque — O(1) Queue CORE

Use deque whenever you need O(1) append/pop from BOTH ends. This is your queue for BFS and your sliding window deque for monotonic problems.

pythondeque — queue and sliding window
from collections import deque

q = deque()

# Queue (FIFO) operations — all O(1)
q.append(1)       # enqueue right
q.append(2)
q.popleft()       # dequeue left → 1   ← this is O(1), list.pop(0) is O(n)!

# Double-ended
q.appendleft(0)   # push to front
q.pop()           # pop from right
q[0]              # peek front
q[-1]             # peek back

# Initialise from iterable
bfs_q = deque([root])   # start BFS with root node

# maxlen — sliding window of fixed size
window = deque(maxlen=3)  # auto-evicts oldest when full

heapq — Priority Queue CORE

Python only has a min-heap via heapq. For a max-heap, negate values. This is essential for top-K problems, Dijkstra, and scheduling.

pythonheapq — min-heap and max-heap trick
import heapq

# Min-heap
h = []
heapq.heappush(h, 3)   # O(log n)
heapq.heappush(h, 1)
heapq.heappush(h, 2)
heapq.heappop(h)        # → 1  (minimum)  O(log n)
h[0]                    # peek min  O(1)

# Convert existing list to heap in O(n)
arr = [5, 3, 1, 4]
heapq.heapify(arr)      # in-place

# Max-heap — negate values
max_h = []
heapq.heappush(max_h, -5)
heapq.heappush(max_h, -1)
max_val = -heapq.heappop(max_h)  # → 5

# Tuples — heap compares by first element
heapq.heappush(h, (dist, node))  # sorted by dist → Dijkstra pattern

# Top-K smallest/largest — very common LeetCode pattern
heapq.nsmallest(3, arr)   # [1, 3, 4]
heapq.nlargest(3, arr)    # [5, 4, 3]

from collections import defaultdict
def group_anagrams(strs):
    groups = defaultdict(list)
    for s in strs:
        key = tuple(sorted(s))   # tuples are hashable, lists aren't
        groups[key].append(s)
    return list(groups.values())

Two Pointers

Covers ~25% of array/string mediums. Master this first.

When to use it

Two pointers means two indices moving through a structure. Use it when: the array is sorted, you're looking for a pair/triplet summing to a target, or you're comparing from both ends (palindrome check, container with most water).

pythontemplate: opposite-end pointers
def two_pointer_template(arr):
    left, right = 0, len(arr) - 1
    while left < right:
        # current = some function of arr[left] and arr[right]
        if condition_met:
            return result
        elif need_larger:
            left += 1     # move left right → increases sum
        else:
            right -= 1   # move right left → decreases sum

# Example: Two Sum II (sorted array) — LeetCode 167
def two_sum_sorted(nums, target):
    l, r = 0, len(nums) - 1
    while l < r:
        s = nums[l] + nums[r]
        if   s == target: return [l, r]
        elif s  < target: l += 1
        else:              r -= 1

# Example: Valid Palindrome
def is_palindrome(s):
    s = [c.lower() for c in s if c.isalnum()]
    l, r = 0, len(s) - 1
    while l < r:
        if s[l] != s[r]: return False
        l += 1; r -= 1
    return True

# Fast/slow pointer — detect cycle (Floyd's)
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow is fast: return True
    return False
TimeO(n)
SpaceO(1)

Sliding Window

Subarray / substring problems. Expand right, shrink left.

Variable-size window template PATTERN

pythontemplate + longest substring without repeating
# Template: find max-length window satisfying a constraint
def sliding_window(s):
    left = 0
    state = {}    # track window contents
    result = 0

    for right in range(len(s)):
        # 1. Expand: add s[right] to window
        state[s[right]] = state.get(s[right], 0) + 1

        # 2. Shrink: while window is invalid, move left
        while state[s[right]] > 1:  # duplicate exists
            state[s[left]] -= 1
            left += 1

        # 3. Update result (window is now valid)
        result = max(result, right - left + 1)

    return result

# Fixed-size window — find max sum of k elements
def max_sum_k(nums, k):
    window = sum(nums[:k])   # initial window
    result = window
    for i in range(k, len(nums)):
        window += nums[i] - nums[i-k]   # slide: add right, remove left
        result = max(result, window)
    return result
TimeO(n)
SpaceO(k)

Binary Search

Know both templates. Then learn "search on answer".

The two templates PATTERN

pythonbinary search — exact match + left boundary
# Template 1: find exact target
def binary_search(nums, target):
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = (lo + hi) // 2   # no overflow in Python — this is always safe
        if   nums[mid] == target: return mid
        elif nums[mid]  < target: lo = mid + 1
        else:                      hi = mid - 1
    return -1

# Template 2: find leftmost position satisfying a condition
# Use when: "find first position where condition is True"
def binary_search_left(nums, target):
    lo, hi = 0, len(nums)   # hi = len (one past end)
    while lo < hi:           # strictly less
        mid = (lo + hi) // 2
        if nums[mid] < target:
            lo = mid + 1
        else:
            hi = mid          # shrink from right
    return lo                 # lo = first index where nums[lo] >= target

# "Search on answer" — binary search on the RESULT SPACE
# Pattern: when you can verify "is X feasible?" in O(n), search for min/max X
def min_days_to_make_bouquets(bloomDay, m, k):
    def feasible(days):
        bouquets, consecutive = 0, 0
        for d in bloomDay:
            if d <= days: consecutive += 1
            else:          consecutive  = 0
            if consecutive == k:
                bouquets += 1; consecutive = 0
        return bouquets >= m

    lo, hi = 1, max(bloomDay)
    while lo < hi:
        mid = (lo + hi) // 2
        if feasible(mid): hi = mid
        else:             lo = mid + 1
    return lo
TimeO(log n)
SpaceO(1)

BFS / DFS

Trees, graphs, islands, shortest paths. Core of every medium graph problem.

BFS — shortest path, level order PATTERN

pythonBFS template — graph and grid
from collections import deque

# Graph BFS
def bfs(graph, start):
    visited = {start}
    q = deque([start])
    while q:
        node = q.popleft()
        for nei in graph[node]:
            if nei not in visited:
                visited.add(nei)
                q.append(nei)

# Level-order BFS (returns levels as lists)
def level_order(root):
    if not root: return []
    result, q = [], deque([root])
    while q:
        level = []
        for _ in range(len(q)):   # snapshot current level size
            node = q.popleft()
            level.append(node.val)
            if node.left:  q.append(node.left)
            if node.right: q.append(node.right)
        result.append(level)
    return result

# Grid BFS (islands, shortest path in maze)
def bfs_grid(grid, sr, sc):
    rows, cols = len(grid), len(grid[0])
    visited = {(sr, sc)}
    q = deque([(sr, sc, 0)])  # (row, col, distance)
    dirs = [(0,1),(0,-1),(1,0),(-1,0)]  # 4-directional
    while q:
        r, c, dist = q.popleft()
        for dr, dc in dirs:
            nr, nc = r+dr, c+dc
            if 0 <= nr < rows and 0 <= nc < cols \
               and (nr,nc) not in visited and grid[nr][nc] != 0:
                visited.add((nr, nc))
                q.append((nr, nc, dist+1))

DFS — recursive and iterative PATTERN

pythonDFS — recursive tree + iterative graph
# Recursive DFS on tree
def dfs_tree(root):
    if not root: return 0   # base case
    left  = dfs_tree(root.left)
    right = dfs_tree(root.right)
    return 1 + max(left, right)  # max depth

# DFS on grid — Number of Islands pattern
def num_islands(grid):
    def dfs(r, c):
        if r < 0 or r >= len(grid) or \
           c < 0 or c >= len(grid[0]) or \
           grid[r][c] != '1': return
        grid[r][c] = '0'      # mark visited (mutate in place)
        for dr, dc in [(0,1),(0,-1),(1,0),(-1,0)]:
            dfs(r+dr, c+dc)
    count = 0
    for r in range(len(grid)):
        for c in range(len(grid[0])):
            if grid[r][c] == '1':
                dfs(r, c)
                count += 1
    return count

# Iterative DFS (avoids Python's 1000-depth stack limit)
def dfs_iterative(graph, start):
    visited = set()
    stack = [start]
    while stack:
        node = stack.pop()
        if node in visited: continue
        visited.add(node)
        for nei in graph[node]:
            if nei not in visited:
                stack.append(nei)
⚠️
Python recursion limit

Python's default recursion limit is ~1000 frames. For deep graphs or large trees, use iterative DFS with an explicit stack, or add import sys; sys.setrecursionlimit(10000) at the top. In interviews, mention you're aware of this.

Dynamic Programming

Top-down with lru_cache first. Bottom-up when you need space optimisation.

Top-down (Memoisation) PATTERN

pythontop-down DP — coin change + knapsack
from functools import lru_cache

# Coin Change — minimum coins to make amount
def coin_change(coins, amount):
    @lru_cache(maxsize=None)
    def dp(rem):
        if rem == 0: return 0
        if rem  < 0: return float('inf')
        return 1 + min(dp(rem - c) for c in coins)
    result = dp(amount)
    return result if result != float('inf') else -1

# Longest Common Subsequence
def lcs(s1, s2):
    @lru_cache(maxsize=None)
    def dp(i, j):
        if i == len(s1) or j == len(s2): return 0
        if s1[i] == s2[j]:
            return 1 + dp(i+1, j+1)
        return max(dp(i+1, j), dp(i, j+1))
    return dp(0, 0)

Bottom-up (Tabulation) PATTERN

pythonbottom-up DP — 1D and 2D table
# Climbing Stairs — 1D DP
def climb_stairs(n):
    if n <= 2: return n
    dp = [0] * (n + 1)
    dp[1] = 1; dp[2] = 2
    for i in range(3, n+1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]

# Space-optimised (rolling vars)
def climb_stairs_opt(n):
    a, b = 1, 2
    for _ in range(3, n+1):
        a, b = b, a+b
    return b

# 2D DP — unique paths in a grid
def unique_paths(m, n):
    dp = [[1] * n for _ in range(m)]  # initialise with 1
    for r in range(1, m):
        for c in range(1, n):
            dp[r][c] = dp[r-1][c] + dp[r][c-1]
    return dp[m-1][n-1]
💡
Strategy: top-down first, optimise later

Always implement top-down with @lru_cache first — it's faster to write and easier to reason about. Only convert to bottom-up if the problem requires O(1) space or you're hitting recursion limits. The examiner sees you get to correct first, then optimise.

Backtracking

Permutations, combinations, subsets, N-queens. One template rules them all.

The universal template PATTERN

pythonbacktracking — subsets, combinations, permutations
# Subsets — choose or skip each element
def subsets(nums):
    result = []
    def backtrack(start, current):
        result.append(current[:])  # snapshot current path
        for i in range(start, len(nums)):
            current.append(nums[i])   # choose
            backtrack(i+1, current)    # explore
            current.pop()              # undo (backtrack)
    backtrack(0, [])
    return result

# Permutations — all orderings
def permutations(nums):
    result = []
    def backtrack(current, remaining):
        if not remaining:
            result.append(current[:])
            return
        for i, n in enumerate(remaining):
            current.append(n)
            backtrack(current, remaining[:i] + remaining[i+1:])
            current.pop()
    backtrack([], nums)
    return result

# Combinations summing to target (with deduplication)
def combination_sum(candidates, target):
    candidates.sort()
    result = []
    def backtrack(start, current, remaining):
        if remaining == 0:
            result.append(current[:]); return
        for i in range(start, len(candidates)):
            if candidates[i] > remaining: break  # pruning
            current.append(candidates[i])
            backtrack(i, current, remaining - candidates[i])
            current.pop()
    backtrack(0, [], target)
    return result
💡
The three-step backtracking mantra

Choose → Explore → Undo. Always append/add before recursing, always pop/remove after. The undo step is what makes it backtracking instead of just DFS. Sort input first to enable pruning with an early break.

import heapq

# Top-K frequent elements
def top_k_frequent(nums, k):
    from collections import Counter
    freq = Counter(nums)
    # heapq.nlargest by frequency — O(n log k)
    return heapq.nlargest(k, freq.keys(), key=freq.get)

# K closest points to origin
def k_closest(points, k):
    # max-heap of size k (negate distance)
    h = []
    for x, y in points:
        dist = x**2 + y**2  # no sqrt needed — monotonic
        heapq.heappush(h, (-dist, x, y))
        if len(h) > k: heapq.heappop(h)  # evict farthest
    return [[x, y] for _, x, y in h]

# Dijkstra shortest path
def dijkstra(graph, src):
    dist = {src: 0}
    h = [(0, src)]   # (distance, node)
    while h:
        d, u = heapq.heappop(h)
        if d > dist.get(u, float('inf')): continue  # stale entry
        for v, w in graph[u]:
            nd = d + w
            if nd < dist.get(v, float('inf')):
                dist[v] = nd
                heapq.heappush(h, (nd, v))
    return dist

# LeetCode provides this — you don't define it
class ListNode:
    def __init__(self, val=0, next=None):
        self.val  = val
        self.next = next

# Reverse a linked list — in-place, O(n) time, O(1) space
def reverse(head):
    prev, curr = None, head
    while curr:
        nxt       = curr.next  # save next
        curr.next = prev       # reverse pointer
        prev      = curr       # advance prev
        curr      = nxt        # advance curr
    return prev

# Dummy head — simplifies edge cases (empty list, head changes)
def merge_two(l1, l2):
    dummy = ListNode(0)
    curr  = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            curr.next, l1 = l1, l1.next
        else:
            curr.next, l2 = l2, l2.next
        curr = curr.next
    curr.next = l1 or l2
    return dummy.next

# Find middle (slow/fast pointer)
def find_middle(head):
    slow = fast = head
    while fast.next and fast.next.next:
        slow = slow.next
        fast = fast.next.next
    return slow   # slow is at middle

# Validate BST — pass bounds down recursively
def is_valid_bst(root, lo=float('-inf'), hi=float('inf')):
    if not root: return True
    if not (lo < root.val < hi): return False
    return (is_valid_bst(root.left,  lo, root.val) and
            is_valid_bst(root.right, root.val, hi))

# Union-Find (Disjoint Set Union) — O(α(n)) ≈ O(1)
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank   = [0] * n

    def find(self, x):
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # path compression
        return self.parent[x]

    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py: return False  # already connected → cycle!
        if self.rank[px] < self.rank[py]: px, py = py, px
        self.parent[py] = px
        if self.rank[px] == self.rank[py]: self.rank[px] += 1
        return True

# Topological sort (Kahn's algorithm — BFS-based)
def topo_sort(n, edges):
    from collections import defaultdict, deque
    graph   = defaultdict(list)
    in_deg  = [0] * n
    for u, v in edges:
        graph[u].append(v); in_deg[v] += 1
    q = deque(i for i in range(n) if in_deg[i] == 0)
    order = []
    while q:
        u = q.popleft()
        order.append(u)
        for v in graph[u]:
            in_deg[v] -= 1
            if in_deg[v] == 0: q.append(v)
    return order if len(order) == n else []  # empty = cycle exists

Tries

Word search, autocomplete, prefix matching.

Clean Python Trie implementation PATTERN

pythonTrie — insert, search, startsWith
class TrieNode:
    def __init__(self):
        self.children = {}    # char → TrieNode
        self.is_end  = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word):
        node = self.root
        for c in word:
            if c not in node.children:
                node.children[c] = TrieNode()
            node = node.children[c]
        node.is_end = True

    def search(self, word) -> bool:
        node = self.root
        for c in word:
            if c not in node.children: return False
            node = node.children[c]
        return node.is_end

    def starts_with(self, prefix) -> bool:
        node = self.root
        for c in prefix:
            if c not in node.children: return False
            node = node.children[c]
        return True  # don't need is_end here

# Alternative: defaultdict-based Trie (more compact)
def make_trie():
    from collections import defaultdict
    return defaultdict(make_trie)   # recursive default factory

def count_bits(n):
    count = 0
    while n:
        n &= n - 1   # clear lowest set bit each iteration
        count += 1
    return count

Merge intervals + meeting rooms PATTERN

pythonmerge intervals + min meeting rooms
# Merge Intervals — sort by start, greedily extend
def merge(intervals):
    intervals.sort(key=lambda x: x[0])
    merged = [intervals[0]]
    for start, end in intervals[1:]:
        if start <= merged[-1][1]:          # overlaps
            merged[-1][1] = max(merged[-1][1], end)
        else:
            merged.append([start, end])
    return merged

# Min meeting rooms — how many rooms needed at peak?
def min_meeting_rooms(intervals):
    starts = sorted(s for s, _ in intervals)
    ends   = sorted(e for _, e in intervals)
    rooms = peak = j = 0
    for s in starts:
        if s < ends[j]: rooms += 1  # new meeting starts before any ends
        else:           j += 1      # reuse room (meeting ended)
        peak = max(peak, rooms)
    return peak
